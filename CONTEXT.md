# 鲁班学派 · 共享语言

Agent 与项目之间的领域术语定义。消除歧义，确保所有角色说同一句话。

## 角色体系

**问道** — 需求求道者。与用户共修需求，从混沌中炼出需求真言。不写代码，不做技术方案。
_Avoid_: 产品经理、需求分析师、PM（用"问道"）

**司南** — Tech Lead。拆需求、定契约、做仲裁。不写代码。
_Avoid_: PM、架构师、技术负责人（用"司南"）

**鲁班** — 执行工程师。按契约编码、方案先行、异常熔断、交证据。
_Avoid_: 程序员、开发者、实现者（用"鲁班"）

**狄公** — 质量守门人。独立验证、三张审查清单、一票否决权。
_Avoid_: QA、测试、审查员（用"狄公"）

**墨子** — 部署运维。构建→测试→部署，Agent 资源管理。
_Avoid_: DevOps、运维、发布者（用"墨子"）

**云笈七签** — 功法总纲（天书）。七签定功法，修哪签，行哪道。不是角色，是功法库。每个角色 SKILL.md 已内嵌完整七签定义。
_Avoid_: 功法切换器、模式切换、配置器（用"云笈"或"云笈七签"）

## 核心标识

**需求真言** — 问道从混沌中炼出的需求结论。含混沌原念、所求之道、筑基之痛、真需求、伪需求（已斩）、未明之境。
_Avoid_: 需求文档、PRD、需求规格（用"需求真言"）

**混沌** — 用户最初表达的模糊念头、感觉、不满。问道的输入材料。
_Avoid_: 需求、想法、点子（用"混沌"）

**三阶炼需求** — 问道的方法论：筑基（痛点寻真）→ 结丹（去伪存真）→ 化神（凝练真言）。
_Avoid_: 需求分析、需求收集（用"三阶炼需求"）

**契约**（《任务拆解契约》）— 司南输出的任务定义文件，含 TASK-ID、DONE、DON'T。鲁班的唯一执行依据。
_Avoid_: spec、需求文档、任务单（用"契约"）

**[榫卯]** — 鲁班交付标记，代表代码已完成、待狄公审查。完美契合，无需返工。
_Avoid_: 交付、完成、done（用"榫卯"）

**[准绳]** — 诊断先行格式。动手前输出 `[准绳] 问题是 ___；证据是 ___；下一步动作是 ___。`
_Avoid_: 计划、方案、分析（用"准绳"）

**[匠心 🔨]** — 超出要求的工匠精神标记。做本分不标记，超预期才标。
_Avoid_: bonus、extra、亮点（用"匠心"）

**三准绳** — 三条不可逾越的行为红线：闭环（贴证据）· 事实驱动（先验证）· 穷尽（C0-C4走完）。
_Avoid_: 原则、规范、要求（用"三准绳" 或 "准绳一/二/三"）

**C0-C4** — 匠心入微五级失败递进：C0起手→C1换方案→C2深究→C3入微→C4极境。
_Avoid_: 重试、再次尝试、换方法（用"C1/C2/..."）

## 流程术语

**放行单**（《质量放行单》）— 狄公审查通过后签发的凭证，墨子部署的前置条件。
_Avoid_: 通过、批准、OK（用"放行单"）

**门禁** — 流程关键节点的硬阻断。司南出契约后必须确认，鲁班无契约必须拒绝，狄公审查不通过必须打回。
_Avoid_: 检查点、审批、review（用"门禁"）

**熔断** — 鲁班3次不同方案失败→输出《阻塞报告》停止编码。狄公同一任务打回3次→升级司南仲裁。
_Avoid_: 放弃、停止、escalation（用"熔断"）

## 七签功法

**问道签**（炼真言）— 筑基→结丹→化神。从混沌中炼出需求真言。心法。
**周天签**（运周天）— Plan→Execute→Verify→Retrospect。循环不息。功法。
**入木签**（深入木）— 5-Why→Evidence→Prevent。公共 Debug 签，任意角色可修习。诀法。
**存真签**（留存真）— Question→Delete→Simplify→Build。去伪存真。式法。
**绳墨签**（弹绳墨）— Keeper Test→Replace/Approve。底线不可违。令法。
**定盘签**（准定盘）— Measure→A/B→Evidence。数据说话。术法。
**契约签**（定契约）— TASK-ID+DONE+DON'T。永远在线的宪法。大法。

**修习语法**: `@角色 +签名` / `@角色 -签名` / `@角色 +签1 +签2`
