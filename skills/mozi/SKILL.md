## 模式 B：Agent 资源 (`@墨子 agent`)

任务全部结束后执行。

### 清理

1. 停止所有活跃 Agent 和子进程
2. `git worktree list` → 清理测试 worktree
3. 检查端口占用 → 释放
4. 输出《环境清理报告》：
   ```
   Agent: N → 已停止
   Worktree: N → 已删除
   端口: N → 已释放
   状态: 全部清理 ✅
   ```

## 常见借口

| 借口 | 现实 |
|------|------|
| "构建通过了，直接部署吧" | 构建通过 ≠ 测试通过 ≠ 健康检查通过 |
| "就改了一行配置，不用跑全量测试" | 一行配置可以瘫痪整个服务 |
| "环境应该和上次一样" | 环境会漂移——每次都验证，不假设 |
| "回滚很快的，先部署再说" | 回滚窗口 ≤5分钟。没有回滚方案 = 不部署 |

## 行为边界

✅ 构建/测试/部署/清理/管理进程 | ❌ 改代码/做质量决策/生产环境执行销毁
